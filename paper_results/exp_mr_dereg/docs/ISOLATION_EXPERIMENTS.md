# RDMA多租户拦截系统 - 性能隔离验证实验设计

## 概述

本文档基于Husky测试套件(NSDI'23)分析，设计了3个性能隔离验证实验(EXP-7~9)，用于验证多租户拦截系统的资源隔离效果。

---

## Husky测试套件适用性分析

### 测试分类

| 类别 | Husky测试 | 适用性 | 原因 |
|------|----------|--------|------|
| BW攻击 | BW-1MB_ib_write_bw.sh | ✅ 高 | 可验证带宽隔离 |
| Cache-QPC | Cache-QPC_ib_write_bw.sh | ✅ 高 | 可验证QP配额限制 |
| Cache-MR | Cache-access_mr_*.sh | ⚠️ 中 | 可验证MR配额，但非硬件隔离 |
| PCIe攻击 | PCIe/*.sh | ❌ 低 | 软件层无法防护PCIe攻击 |
| PU攻击 | PU/*.sh | ❌ 低 | 软件层无法防护PU攻击 |

### 核心结论

Husky测试套件主要验证**硬件级**RDMA资源隔离，而我们的系统是在**软件层**通过LD_PRELOAD拦截实现隔离。因此：
- 适用于我们的测试：带宽竞争、QP/MR数量限制
- 不适用于我们的测试：PCIe竞争、网卡内部缓存竞争

## 设计的实验 (EXP-7 ~ EXP-9)

### EXP-7: 带宽隔离验证 (Bandwidth Isolation)

**目的**: 验证一个租户的高带宽传输不会影响另一租户

**测试方法**:
```
┌─────────────────┐                    ┌─────────────────┐
│   guolab-8      │◄─────RDMA─────────►│    guolab-6     │
│ 192.168.108.2   │      (100Gbps)      │ 192.168.106.2   │
├─────────────────┤                    ├─────────────────┤
│ 租户10 (Victim) │                    │ 租户10 (Victim) │
│ 租户20 (Attack) │                    │ 租户20 (Attack) │
└─────────────────┘                    └─────────────────┘
```

**评估指标**:
- 基线带宽: Victim单独运行时的带宽
- 干扰带宽: Victim+Attacker同时运行时的带宽
- 隔离度 = 干扰带宽 / 基线带宽

**预期结果**: 隔离度 ≥ 95%

**文档**: [exp7_isolation_bw/README.md](exp7_isolation_bw/README.md)

---

### EXP-8: QP资源隔离验证 (QP Resource Isolation)

**目的**: 验证一个租户的大量QP创建不会阻塞另一租户

**测试方法**:
| 场景 | Victim (租户10) | Attacker (租户20) |
|------|----------------|------------------|
| 基线 | 创建10个QP | 空闲 |
| 并发 | 创建10个QP | 尝试创建100个QP |
| 达限后 | 创建10个QP | 已达配额上限 |

**评估指标**:
- Attacker限制率 = (100-实际成功)/100
- Victim成功率 = Victim成功数/10
- 延迟影响 = (并发延迟-基线延迟)/基线延迟

**预期结果**: 
- Attacker限制率 ≥ 80% (只成功20个)
- Victim成功率 = 100%
- 延迟影响 < 20%

**文档**: [exp8_isolation_qp/README.md](exp8_isolation_qp/README.md)

---

### EXP-9: MR资源隔离验证 (MR Resource Isolation)

**目的**: 验证一个租户的大量MR注册不会阻塞另一租户

**测试方法**: 同EXP-8，测试对象改为MR

**评估指标**: 同EXP-8

**预期结果**:
- Attacker限制率 ≥ 50% (只成功50个)
- Victim成功率 = 100%
- 延迟影响 < 20%

**文档**: [exp9_isolation_mr/README.md](exp9_isolation_mr/README.md)
